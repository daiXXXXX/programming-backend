# 编辑器输入样式异常二次修复说明

## 问题说明

第一次修复后，代码输入框仍然存在样式异常，说明问题并不只是 Monaco 的主题配置，而是编辑器本身在当前页面环境中的渲染稳定性不足。

## 本次处理

本次将工作台中的可编辑代码输入框改为更稳定的自定义文本编辑实现：

- 文件：`programming-frontend/src/components/CodeEditor.tsx`
- 采用深色代码输入区样式
- 保留等宽字体
- 增加行号显示
- 支持滚动同步
- 移除对 Monaco 可编辑实例的依赖，避免白屏和输入样式错乱

同时清理了不再需要的 Monaco 专用全局覆盖样式：

- 文件：`programming-frontend/src/app/globals.css`

## 调整原因

- Monaco 在当前工作台编辑场景下仍然存在渲染不稳定问题
- 用户核心诉求是“可正常输入、显示正常、样式稳定”
- 自定义文本编辑实现更简单、可控、稳定，能直接消除白屏与异常排版问题

## 当前效果

修复后，工作台代码框应具备：

- 正常显示
- 正常输入
- 正常的等宽代码字体
- 稳定的深色背景
- 行号与代码滚动同步

## 影响范围

本次只影响工作台中的代码输入组件，不影响提交详情等只读代码展示页面。
